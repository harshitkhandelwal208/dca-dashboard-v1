Start Here

The DCA Bot Suite has two parts: the Discord bot and the dashboard website.

Discord bot

Handles slash commands, prefix commands, recruitment threads, reaction-role sync, reminders, moderation actions, feed announcements, and spreadsheet reports.

Dashboard

Controls persistent configuration. Use it for channels, roles, panels, spreadsheets, feeds, team counts, logs, and server settings.

Data flow

Dashboard settings are saved to config, bot utilities read that config, and Discord commands or scheduled jobs execute workflows.

Dashboard accessDiscord OAuth plus the configured dashboard access role.
Spreadsheet accessAdministrator permission or the configured spreadsheet team access role.
Recruitment accessRecruiter and manager role settings control ticket operations.
Community vs recruitmentThe dashboard distinguishes community and recruitment server channels/roles so features post to the right guild.

Dashboard Operations

Use the dashboard as the main configuration surface.

Save before relying on changes

Most edits stay local until Save is pressed. After saving, use the relevant sync button for Discord messages that need to be refreshed.

Recruitment Tickets

Recruitment workflows cover panel setup, applicant threads, staff actions, tutorials, transcripts, and outcomes.

Panel

Configure panel channel, title, description, questions, private thread mode, recruiter role, and log channels.

Ticket lifecycle

Applicant opens a ticket, staff claims it, staff reviews answers/screenshots, staff closes with an outcome, and the bot archives/locks the thread.

Records

Recruitment logs, transcript previews, ticket metadata, and applicant images are visible from the Logs dashboard tab.

Team Event Spreadsheets

Spreadsheet sessions convert HCR2 event screenshots into structured outputs and reports.

Event output

Processed events produce an XLSX workbook, an own-team spreadsheet image, and a chart image. Local image copies are removed after Discord upload or by retention cleanup.

Period reports

Weekly reports focus on blue kills and KAB. Monthly reports roll up score, points, attendance, missed events, blue kills, and KAB.

Report FieldMeaning
BluesNumber of enemy players a driver ranked above.
%KillTotal blues killed divided by total possible blues.
TotalTotal blues killed in the report period.
MaxTotal possible blues in the report period.
#KABEvents where the driver ranked above every opponent.
/spreadsheets status / sessionsInspect setup and recent sessions.
/spreadsheets generate rerun_gemini:trueProcess a session and optionally rerun Gemini extraction.
/spreadsheets summary / file / chartReview parsed data or retrieve generated artifacts.
/spreadsheets weekly / monthly anchor_date:YYYY-MM-DDBuild period reports for a specific week or month.
/spreadsheets correctApply a generic field correction when a staff member needs one command for names, teams, ranks, points, scores, or event names.
/spreadsheets correct-name / correct-team / correct-placement / correct-points / correct-event-nameFix OCR or classification mistakes and rebuild outputs.
/spreadsheets rebuild / regenerate-weekly / regenerate-monthlyRegenerate event files or force report posting.
/spreadsheets force-weekly / force-monthlyTemporary staff commands for forcing report posts during production verification.
Best practice

Post all screenshots for one event together, from one user, in the monitored channel. Do not mix unrelated images during the grouping window.

Storage behavior

Source screenshots are temporary processing inputs. Generated spreadsheet/report images are posted to Discord and then deleted locally; any leftovers are removed by the configured local image retention window.

Reaction Roles

Reaction role panels assign community server roles through configured emoji options.

Panel

Each panel has a name, enabled state, channel, message ID, message text, and role options.

Options

Each option maps an emoji to a role and can include a human-readable label.

Sync

After editing panels, sync reaction roles so Discord messages and reactions match the dashboard.

/roles addAdd a role to a user.
/roles removeRemove a role from a user.
